NetWatch Network Connection Monitor

Overview

NetWatch is a network connection monitoring tool for Windows that displays active TCP and UDP connections with detailed process and security context information. It provides real-time monitoring of network endpoints with a focus on security-relevant metadata.

Note: NetWatch requires administrative privileges to display connection statistics and integrity levels.

Getting Started

System Requirements

  • Windows 10 or later (x64)
  • Administrator privileges (recommended for full functionality)

Launching NetWatch

NetWatch can be launched from the command line or by double-clicking the executable.

Command-line syntax:

netwatch.exe [options]

Options:

  • --filter <name> or -f <name> - Filter connections by process name
  • --help or -h or /? - Display help information

Examples:

netwatch.exe --filter chrome
netwatch.exe -f svchost

Toolbar

The toolbar provides quick access to common functions:

  • Refresh - Manually refresh the connection list
  • Pause - Pause automatic refreshing
  • End Process - Terminate the selected process
  • Close Connection - Close the selected TCP connection
  • About - Display program information

Menu Reference

File Menu

  • Refresh Now (F5) - Manually refresh the connection list
  • Pause (Space) - Pause or resume automatic refreshing
  • Exit - Close NetWatch

Options Menu

  • Always On Top - Keep the NetWatch window on top of other windows
  • Show Unconnected Endpoints - Display listening sockets and unconnected UDP endpoints
  • Filter... (Ctrl+F) - Set a process name filter

Process Menu

  • End Process... (Delete) - Terminate the selected process
  • Close Connection - Close the selected TCP connection
  • Properties - View detailed properties of the selected connection
  • Whois - Perform a Whois lookup on the remote IP address
Warning: Terminating processes or closing connections may cause applications to stop working or lose unsaved data. Use these features with caution.

View Menu

  • Toolbar - Show or hide the toolbar
  • Status Bar - Show or hide the status bar

Help Menu

  • Help (F1) - Display this help file
  • About NetWatch - Display version and copyright information

Filtering Connections

You can filter the displayed connections by process name using one of these methods:

  • Launch NetWatch with the --filter or -f command-line option
  • Select Options → Filter... from the menu (or press Ctrl+F)

The filter performs a case-insensitive substring match on process names. Leave the filter blank to show all processes.

Understanding Integrity Levels

Windows uses integrity levels as part of its security model. NetWatch displays the integrity level of each process:

  • System - System-level processes (highest privilege)
  • High - Elevated/administrator processes
  • Medium - Standard user processes
  • Low - Sandboxed or restricted processes
Note: Integrity levels are only available when NetWatch is run with administrative privileges.

Connection States

TCP connections can be in various states:

  • LISTENING - Waiting for incoming connections
  • ESTABLISHED - Active connection
  • CLOSE_WAIT - Waiting for connection termination
  • TIME_WAIT - Waiting after connection close
  • SYN_SENT - Connection attempt in progress
  • SYN_RECEIVED - Connection request received
  • FIN_WAIT_1 - Connection terminating
  • FIN_WAIT_2 - Connection terminating
  • CLOSING - Connection terminating
  • LAST_ACK - Final acknowledgment

UDP is a connectionless protocol and does not have connection states.

Privacy and Security

NetWatch reads network connection information from the Windows operating system. It does not send any data over the network or store any data outside of your computer.
